How things work

Drivers, Executors, and Partitions

Drivers orchestrate and collect all the results back together at the end

There is typically one driver node

Executors actually carry out the tasks

There will usually be many executors and the driver node tells all of them what to do

Partitions are the chunks of data that tasks get performed on

The data will already have been paritioned before our job

Problem

Let’s suppose we want a new table which shows how much money we have made off each product.

  • Join the two tables on PRODUCT_ID
  • Group the new table by PRODUCT_ID
  • Aggregate by summing up
  • Collect this new table into
from pyspark.sql import functions as F

orders = spark.table("fct_orders")
products = spark.table("dim_products")

result = (
    orders
    .join(products, orders.product_id == products.id, "inner")
    .groupBy("product_id")
    .agg(
        F.sum("COST").alias("total_amount")
    )
    .collect()
)
